# AI Spend Audit - Project Reflection

## Overview
This document reflects on the development experience of the AI Spend Audit project.

---

## What I Learned
- How to build a full-stack project using Node.js and frontend technologies
- How frontend and backend communicate using APIs
- How to structure a project into modular components
- Basics of data handling using JSON
- Importance of clean project documentation

---

## Challenges Faced
- Understanding backend request flow
- Managing multiple project files properly
- Structuring data for meaningful output
- Connecting frontend with backend API correctly

---

## Improvements I Want to Make
- Add database instead of JSON storage
- Improve UI with better design system
- Add charts and visual analytics
- Deploy project on cloud platform
- Improve error handling in backend

---

## Key Takeaways
- Small projects can be structured like real products
- Clean architecture makes scaling easier
- Documentation is as important as code
- Backend-frontend separation improves clarity

---

## Overall Experience
This project helped me understand real-world full-stack development workflow and how simple tools can be combined to build useful applications.
